正则表达式

|Word count:2.6k|Reading time:8min|Post View:

注：使用 python 正则表达式库re实现，部分是由 chatGPT 实现

元字符

下面是一些常见的正则表达式元字符及其含义的列表：

元字符 含义
. 匹配除换行符以外的任意字符
^ 匹配字符串的开头
$ 匹配字符串的结尾
* 匹配前面的元素零次或多次
+ 匹配前面的元素一次或多次
? 匹配前面的元素零次或一次
{n} 匹配前面的元素恰好 n 次
{n,} 匹配前面的元素至少 n 次
{n,m} 匹配前面的元素至少 n 次，但不超过 m 次
[…] 匹配方括号内的任意一个字符
[^…] 不匹配方括号内的任意一个字符
\d 匹配任意一个数字字符
\D 匹配任意一个非数字字符
\w 匹配任意一个字母、数字或下划线字符
\W 匹配任意一个非字母、非数字且非下划线字符
\s 匹配任意一个空白字符（包括空格、制表符、换行符等）
\S 匹配任意一个非空白字符
\b 匹配单词的边界
\B 匹配非单词的边界
(…) 创建捕获组
| 匹配两个或多个模式之一
\ 转义字符，用于匹配特殊字符或将特殊字符按字面意义匹配

这些元字符在正则表达式中具有特殊的含义，用于定义模式的匹配规则。通过结合这些元字符以及其他正则表达式的功能，可以构建灵活且强大的模式来满足各种匹配需求。

分组语法

正则表达式的分组语法允许将模式的一部分组合在一起，并对其进行操作。下表列出了正则表达式中常用的分组语法：

分组语法 含义
(pattern) 创建一个普通捕获组
(?:pattern) 创建一个非捕获组
(?P<name>pattern) 创建一个命名捕获组，使用名称引用捕获的内容
(?P=name) 引用之前命名捕获组中匹配的内容
(?#comment) 创建一个注释，用于在正则表达式中添加注释
(?=pattern) 正向肯定预查，要求模式后面的内容匹配 pattern
(?!pattern) 正向否定预查，要求模式后面的内容不匹配 pattern
(?<=pattern) 反向肯定预查，要求模式前面的内容匹配 pattern
(?<!pattern) 反向否定预查，要求模式前面的内容不匹配 pattern

使用分组语法，可以对正则表达式中的一部分模式进行分组，并对分组内的内容进行特定操作。例如，捕获组可以用于提取特定的子字符串，预查可以用于限定匹配的条件，注释可以提供模式的解释。
以下是一个示例，说明了正则表达式中分组语法的使用：

import re

pattern = r'(ab)+'
text = "abababab"

# 使用捕获组提取匹配的内容
matches = re.findall(pattern, text)
print("捕获组匹配结果:", matches)

# 使用预查限定匹配条件
result = re.search(r'(?=ab)ab+', text)
if result:
    print("正向肯定预查匹配成功:", result.group())

# 使用注释解释模式
result = re.search(r'ab+  # 匹配一个或多个 ab', text, re.X)
if result:
    print("注释解释模式匹配成功:", result.group())

在上述示例中，我们使用了不同的分组语法。捕获组 (ab)+ 用于提取连续的 “ab” 子字符串，正向肯定预查 (?=ab)ab+ 用于要求匹配的子字符串以 “ab” 开头，注释 ab+ # 匹配一个或多个 ab 用于提供模式的解释。
通过合理使用分组语法，可以更好地控制正则表达式的匹配行为，并提高匹配的准确性和灵活性。

懒惰模式

注：贪婪模式是默认模式
懒惰限定词（Lazy Quantifiers）是正则表达式中用于实现懒惰匹配的特殊语法。下表列出了常见的懒惰限定词及其含义：

懒惰限定词 含义
*? 懒惰匹配，匹配前面的元素零次或多次，尽可能少
+? 懒惰匹配，匹配前面的元素一次或多次，尽可能少
?? 懒惰匹配，匹配前面的元素零次或一次，尽可能少
{n,m}? 懒惰匹配，匹配前面的元素至少 n 次，但不超过 m 次，尽可能少

使用懒惰限定词，可以指示正则引擎在匹配时尽量选择最短的匹配。懒惰限定词与贪婪限定词（如 *、+、? 和 {n,m}）具有相同的含义和功能，只是在其后添加 ? 来表示懒惰匹配。
以下是一个示例，展示了懒惰限定词的应用：

import re

text = "abbbb"

# 使用懒惰限定词进行懒惰匹配
lazy_match = re.search(r'a.*?b', text)
if lazy_match:
    print("懒惰匹配结果:", lazy_match.group())

# 使用贪婪限定词进行贪婪匹配
greedy_match = re.search(r'a.*b', text)
if greedy_match:
    print("贪婪匹配结果:", greedy_match.group())

在上述示例中，我们使用正则表达式模式 a.*?b 进行懒惰匹配，以及 a.*b 进行贪婪匹配。对于输入字符串 “abbbb”，懒惰匹配会尽量少地匹配字符，因此匹配结果为 “ab”。而贪婪匹配则会尽可能多地匹配字符，导致匹配结果为整个字符串 “abbbb”。
懒惰限定词在需要匹配最短可能结果的情况下非常有用。通过使用适当的限定词，我们可以灵活地控制正则表达式的匹配行为，以满足不同的需求。

后向引用

后向引用（Backreferences）是正则表达式中一种强大的功能，它允许在模式中引用先前捕获的内容。通过后向引用，可以实现对重复的模式进行匹配和提取。
后向引用使用捕获组（Capture Groups）来标记和引用先前捕获的内容。捕获组用圆括号 ( ) 包围，并通过编号或名称进行引用。

以下是后向引用的使用示例：

import re

# 使用后向引用匹配重复的单词
text = "apple apple orange orange banana"
pattern = r"\b(\w+)\s+\1\b"

matches = re.findall(pattern, text)
print(matches)

在上述示例中，我们使用正则表达式模式 r"\b(\w+)\s+\1\b" 来匹配重复的单词。该模式中的 \b 表示单词边界，(\w+) 表示一个捕获组，用于匹配一个或多个单词字符，\s+ 表示一个或多个空白字符，\1 是后向引用，引用第一个捕获组的内容，\b 表示单词边界。因此，整个模式会匹配重复的单词。
运行上述代码，输出结果为 ['apple', 'orange']，表示成功匹配到了重复的单词 “apple” 和 “orange”。
后向引用的语法如下：

  • 使用编号引用：\n，其中 n 是捕获组的编号，从 1 开始。
  • 使用命名引用：(?P=name)，其中 name 是捕获组的名称。

通过后向引用，我们可以更灵活地处理正则表达式中的重复模式，并提取或替换特定的重复内容。

零宽断言

零宽断言（Zero-width assertions）是正则表达式中一种强大的技术，用于在匹配时指定特定的位置而不消耗字符。零宽断言可以用来限定匹配的上下文，而不会实际匹配或消耗这些上下文中的字符。
常见的零宽断言包括正向前视断言（Positive Lookahead）、负向前视断言（Negative Lookahead）、正向后顾断言（Positive Lookbehind）和负向后顾断言（Negative Lookbehind）。

以下是零宽断言的使用示例：

import re

# 使用正向前视断言匹配包含 "apple" 的单词
text = "I like apples and oranges"
pattern = r"\w+(?= apple)"

matches = re.findall(pattern, text)
print(matches)

在上述示例中，我们使用正则表达式模式 r"\w+(?= apple)"，其中 (?= apple) 是正向前视断言，表示要匹配的单词后面必须紧跟着 “ apple”。\w+ 匹配一个或多个单词字符。因此，整个模式会匹配包含 “ apple” 的单词。
运行上述代码，输出结果为 ['like']，表示成功匹配到了包含 “ apple” 的单词 “like”。
其他零宽断言的语法如下：

  • 负向前视断言：(?!pattern)，表示在当前位置后面不能匹配 pattern。
  • 正向后顾断言：(?<=pattern)，表示当前位置前面必须匹配 pattern。
  • 负向后顾断言：(?<!pattern)，表示当前位置前面不能匹配 pattern。

通过零宽断言，我们可以更精确地定义匹配的条件，而不需要实际消耗匹配的字符。这在某些场景下非常有用，例如查找特定模式的单词、限定上下文等。

POSIX 字符类

以下是一些常见的 POSIX 字符类及其对应的含义：
注：在 Linux 字符处理命令中可用，但在正则表达式中并不通用。

POSIX 字符类 含义
[:alnum:] 匹配任意字母和数字字符（相当于 [A-Za-z0-9]）
[:alpha:] 匹配任意字母字符（相当于 [A-Za-z]）
[:blank:] 匹配空格和制表符字符
[:cntrl:] 匹配控制字符
[:digit:] 匹配任意数字字符（相当于 [0-9]）
[:graph:] 匹配任意可打印字符（相当于 [!-~]，不包括空格）
[:lower:] 匹配任意小写字母字符（相当于 [a-z]）
[:print:] 匹配任意可打印字符（相当于 [ -~]，包括空格）
[:punct:] 匹配任意标点字符
[:space:] 匹配任意空白字符（包括空格、制表符、换行符等）
[:upper:] 匹配任意大写字母字符（相当于 [A-Z]）
[:xdigit:] 匹配任意十六进制数字字符（相当于 [0-9A-Fa-f]）
[:word:] 匹配任意单词字符（字母、数字、下划线，相当于 [A-Za-z0-9_]）
[:ascii:] 匹配 ASCII 字符集中的字符

这些 POSIX 字符类可以在正则表达式中使用，以表示特定类别的字符。例如，[:digit:] 表示数字字符，[:alpha:] 表示字母字符，[:space:] 表示空白字符等。
需要注意的是，不同的正则表达式引擎对 POSIX 字符类的支持程度可能会有所差异，因此在使用时请参考具体的正则表达式引擎文档以确保兼容性。
